🛡️ SAFETY GUARANTEE — your base branch is NEVER touched
git stash — saves your uncommitted files locally. Does not touch any branch.
git fetch origin — downloads info only. Does not change any file. Does not touch develop.
git reset --soft origin/mvp/develop — only moves YOUR feature branch pointer back. develop is read-only here, used only as a reference point. It is NEVER modified.
git rebase origin/mvp/develop — replays YOUR commits on top of develop. Only your feature branch moves. develop is untouched.
git push origin mvp/features/your-branch --force-with-lease — pushes ONLY your feature branch. develop on origin is completely unaffected.
✓ Every single command in this guide operates exclusively on your feature branch.
✓ origin/mvp/develop is used only as a read-only reference point — never written to, never modified.
✓ To accidentally push to develop you would have to explicitly run git push origin mvp/develop — which we never do.
✅ THE ONLY CORRECT ORDER — memorize this
1. STASH
→
2. FETCH
→
3. SQUASH
→
4. REBASE
→
5. PUSH --force-with-lease
→
6. STASH POP
Every single time. Before every PR. Before every push after rebase. This order never changes.
⚠️ Never skip squash. If you rebase with 5 commits, you get up to 5 conflict stops. Squash first = max 1.
⚠️ Never skip rebase. Squash without rebase means your commit sits on old develop. Rebase puts it on the latest tip.
🚫 STRICTLY BANNED — never use these commands or buttons
git pull MOST DANGEROUS
Secretly runs git fetch + git merge. Creates hidden merge commit. Pollutes history forever. Makes PR unreadable. Completely skips squash AND rebase.
✅ INSTEAD — do all steps:
git stash
git fetch origin
git reset --soft origin/develop   # squash
git add . && git commit -m "feat: your feature"
git rebase origin/develop
git add . && git commit -m "feat: your feature"   # if conflict arose during rebase
git push origin branch --force-with-lease
git stash pop
VS Code "Sync Changes" button EXTREMELY DANGEROUS
Does git pull + git push in ONE CLICK silently. Skips squash entirely. Skips rebase entirely. Pushes all your WIP/fix/test commits raw.
✅ Instead → Use terminal (Ctrl+`). Run stash → fetch → squash → rebase → push → stash pop manually.
VS Code "Pull" button (↓ arrow)
Identical to git pull. Same hidden merge. Same skipped squash and rebase. Never click it.
✅ Instead → terminal: stash → fetch → squash → rebase → push → stash pop
git push --force
Overwrites remote without checking. Silently deletes teammates' commits. No warning. No recovery.
✅ Instead → git push --force-with-lease (checks first, fails safely)
git reset --hard
Permanently deletes ALL uncommitted file changes. Cannot undo. Hours of work gone.
✅ Instead → git stash (saves work) or git reset --soft HEAD~1 (removes commit, keeps files)
VS Code "Discard All Changes"
Wipes ALL uncommitted work permanently. No recovery.
✅ Instead → git stash to save, or discard individual files only
Rebase without squashing first COMMON MISTAKE
If you have 5 commits and run git rebase origin/develop WITHOUT squashing first, Git replays each commit one by one. Each one can cause a separate conflict. Squash first → 1 commit → max 1 conflict.
✅ Always squash first: git reset --soft origin/develop → git add . → git commit → THEN rebase
✅ What to do — the correct workflow always
1
Start of day / before any work — git fetch origin to see what changed. Never pull. Never sync.
2
Before raising PR — stash → fetch → squash → rebase → push → stash pop. Always squash BEFORE rebase.
3
Creating a new branch — fetch first → checkout base → rebase base → checkout -b new-name
4
Switching branches — stash or commit first, then fetch, then checkout. Never click sync.
5
Resolving conflicts during rebase — fix file → git add . → git rebase --continue
6
Stuck after clicking Pull/Sync? — Abort → stash → reset → start fresh. See 🆘 Rescue in Step Guide.

📋 All Commands

What every command does — correct workflow only

Moving around

# see current branch and changes
$ git status
# switch to a branch
$ git checkout branch-name
# create new branch and switch
$ git checkout -b feature/my-feature
# see all branches
$ git branch -a
# see commit history
$ git log --oneline
# see remote URL
$ git remote -v

Saving your work

# stage all changed files
$ git add .
# stage one file
$ git add src/login.js
# see what changed before adding
$ git diff
# commit
$ git commit -m "feat: add login page"

Stash — save temporarily

# stash everything uncommitted
$ git stash
# bring back
$ git stash pop
# see stashes
$ git stash list
# throw away stash
$ git stash drop

The correct full flow (stash → fetch → squash → rebase → push → stash pop)

# 0. stash uncommitted files
$ git stash
# 1. download latest (safe — does not touch develop)
$ git fetch origin
# 2. squash all commits into one (develop used as reference only)
$ git reset --soft origin/develop
$ git add .
$ git commit -m "feat: your feature"
# 3. rebase on top of latest develop tip (develop not touched)
$ git rebase origin/develop
# if conflict: fix file → git add . → git rebase --continue
# 4. force push your feature branch only
$ git push origin your-branch --force-with-lease
# 5. restore stashed files
$ git stash pop

Squash — 3 methods

Method 1 — Soft reset (easiest, recommended)

$ git reset --soft origin/develop
$ git add .
$ git commit -m "feat: your feature"
✓ all work in 1 commit. No editor. develop untouched.

Method 2 — Interactive rebase (no counting)

$ git rebase -i origin/develop
# first line "pick", rest change to "s", save, write message, save

Method 3 — Interactive rebase (with count)

# count commits: git log --oneline
$ git rebase -i HEAD~4
# same: first "pick", rest "s", save, message, save

Cherry pick

$ git log --oneline origin/their-branch
$ git checkout your-branch
$ git cherry-pick d4e5f6
# conflict? fix → git add . → git cherry-pick --continue
# cancel: git cherry-pick --abort

Undoing things

✅ Soft reset (safest)
Removes commit. Keeps files. develop untouched.
$ git reset --soft HEAD~1
$ git reset --soft origin/develop
🔵 Revert (safe for shared)
New commit that undoes. No history rewrite.
$ git revert HEAD
$ git push origin your-branch

Conflicts

<<<<<<< HEAD
your code
=======
their code
>>>>>>> origin/develop
Pick one or combine. Delete ALL markers. Then:
$ git add .
$ git rebase --continue
# cancel: git rebase --abort
